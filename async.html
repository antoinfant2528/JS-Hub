Asynchronous JavaScript

Asynchronous JavaScript allows you to perform tasks without blocking the main thread, enabling smoother user experiences. Common asynchronous patterns include callbacks, promises, and async/await.

Learn More ↓

Callbacks

A Callback is a function passed as an argument to another function, which is executed after a certain event or operation is completed. 

function fetchData(callback) {
    setTimeout(() => {
      const data = 'Sample Data';
      callback(data);
    }, 1000);
  }
  fetchData((data) => {
    console.log(data); // Output after 1 second: Sample Data
  });

Promises

A promise is an object that represents the eventual completion (or failure) of an asynchronous operation and its resulting value. 

const fetchData = new Promise((resolve, reject) => {
    setTimeout(() => {
      const data = 'Sample Data';
      resolve(data);
    }, 1000);
  });
  fetchData.then((data) => {
    console.log(data); // Output after 1 second: Sample Data
  });

Async/Await

Async/await is syntactic sugar built on top of promises, allowing you to write asynchronous code that looks synchronous. 

function fetchData() {
    return new Promise((resolve, reject) => {
      setTimeout(() => {
        const data = 'Sample Data';
        resolve(data);
      }, 1000);
    });
  }
  async function getData() {
    const data = await fetchData();
    console.log(data); // Output after 1 second: Sample Data
  }
  getData();
