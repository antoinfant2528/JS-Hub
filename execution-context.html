Execution Context

Understand how JavaScript runs your code — from global to function execution contexts and the call stack.

Learn More ↓

What is Execution Context?

The Execution Context is the environment in which JavaScript code is executed. It determines which variables, functions, and objects are accessible at a particular time during program execution.

Types of Execution Context

  • Global Execution Context (GEC): Created automatically when a JS program starts. It handles all global code.
  • Function Execution Context (FEC): Created each time a function is invoked.
  • Eval Execution Context: Created when code runs inside eval().

Phases of Execution Context

  1. Creation Phase: Variables are allocated memory and initialized with undefined. Functions are fully hoisted.
  2. Execution Phase: The code runs line by line, replacing undefined with actual values.

Example

var greeting = "Hello";

      function greet() {
        var name = "Anto";
        console.log(greeting + ", " + name);
      }

      greet();

1️⃣ A Global Execution Context is created first.
2️⃣ The Creation Phase allocates memory for greeting and greet().
3️⃣ When greet() is called, a Function Execution Context is created and pushed to the Call Stack.
4️⃣ After execution, it’s popped from the stack.
