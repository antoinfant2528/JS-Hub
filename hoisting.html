Hoisting

Learn how JavaScript moves variable and function declarations to the top of their scope.

Learn More ↓

Understanding Hoisting

Hoisting is JavaScript’s default behavior of moving variable and function declarations to the top of the current scope before code execution.

Example with Variables

console.log(message); // undefined
var message = "Hello, JS!";

Here, JavaScript internally does this:

var message;
console.log(message);
message = "Hello, JS!";

Hoisting in Functions

sayHello();            
function sayHello() {
  console.log("Hi there!");
}

Function declarations are fully hoisted — so you can call them before they’re defined.

Let & Const Are Different

console.log(a); // ReferenceError
let a = 5;

let and const are hoisted but stay in the Temporal Dead Zone until their line of declaration executes.
