JavaScript Scope Explained

Scope determines the visibility or accessibility of variables. It defines where variables can be used in your code.

1. Global Scope

Variables declared outside any function or block are globally scoped — accessible anywhere in the script. 

var name = "Anto";

function greet() {
  console.log(name); // Accessible
}

greet();

2. Function Scope

Variables declared inside a function are accessible only within that function. 

function test() {
  var message = "Hello World";
  console.log(message);
}
console.log(message); // ❌ Error - not defined

3. Block Scope (let & const)

With let and const, JavaScript introduced block scope — variables exist only inside curly braces {}. 

{
  let a = 10;
  const b = 20;
}
console.log(a); // ❌ ReferenceError

4. Lexical Scope

Inner functions can access variables of their outer functions — this is called lexical scoping. 

function outer() {
  let outerVar = "Outer";

  function inner() {
    console.log(outerVar); // ✅ Works!
  }

  inner();
}
